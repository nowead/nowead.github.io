## 1. Vulkan이란?

Vulkan은 **그래픽스 API**다.

### API가 뭘까?

**API**(Application Programming Interface)는 두 소프트웨어가 서로 대화할 수 있게 해주는 중개자다.

**그래픽스 API**는 애플리케이션이 그래픽 관련 작업을 할 수 있게 해주는 인터페이스로, 더 쉽게 말하면 "**내 코드가 GPU에게 명령을 내릴 때 사용하는 문법 및 규칙**"이다.

### 다른 그래픽스 API는?

널리 알려진 그래픽스 API에는 **OpenGL, DirectX, Vulkan, Metal** 등이 있다.
모두 그래픽 카드(GPU)에게 "화면에 그림을 그려줘"라고 명령을 내리는 도구들이지만, 접근 방식과 제어 수준에서 큰 차이가 있다.

가장 큰 차이점은 다음 두 가지다:

- **얼마나 개발자가 직접 챙겨야 하는가** (난이도)
- **어디서 실행되는가** (호환성)

### 주요 그래픽스 API 비교

| 특징 | OpenGL | DirectX (11 / 12) | Vulkan |
| :---------- | :-------------------- | :------------------------------ | :---------------------------- |
| **난이도** | 쉬움 (입문 추천) | 중간 (DX11) / 어려움 (DX12) | 매우 어려움 (고수용) |
| **제어권** | 낮음 (자동화 많음) | 중간 / 높음 | 매우 높음 (수동 제어) |
| **성능 잠재력** | 보통 | 높음 | 매우 높음 |
| **플랫폼** | 크로스 플랫폼 | Windows, Xbox 전용 | 크로스 플랫폼 |
| **주요 용도** | 그래픽 입문, 연구, 교육용 | PC/콘솔 게임 표준 | 고성능 게임 엔진, 모바일 |

> **참고**: Mac은 현재 OpenGL 지원을 중단하고 자체 API인 **Metal**을 밀고 있다.

## 2. 각 API 상세 분석

### 2.1. OpenGL (Legacy & Universal)

- **특징**: 가장 역사가 깊고 범용적인 API다. "일단 화면에 뭔가를 띄우는 것"이 목표라면 가장 빠르고 쉽다.

- **장점**:
  - 수많은 예제와 자료가 있어 배우기 쉽다
  - 윈도우, 리눅스 가리지 않고 대부분의 플랫폼에서 동작한다
  - 초보자가 그래픽 프로그래밍을 배우기에 이상적이다

- **단점**:
  - 오래된 설계로 최신 GPU 성능을 100% 활용하기 어렵다
  - 드라이버가 백그라운드에서 많은 작업을 자동으로 처리하여, 성능 예측과 최적화가 어렵다
  - 멀티스레드 지원이 약하다

### 2.2. DirectX (Industry Standard)

- **특징**: 마이크로소프트가 만든 윈도우 게임 개발의 표준이다. 그래픽뿐만 아니라 사운드, 입력 장치 제어까지 포함된 종합 멀티미디어 API다.

- **버전별 차이**:
  - **DirectX 11**: OpenGL처럼 다루기 쉽고 생산성이 좋아 아직도 많은 게임에서 사용된다
  - **DirectX 12**: Vulkan처럼 Low-level API로 바뀌어 성능을 극대화할 수 있다

- **장점**:
  - 게임 업계 표준으로 가장 많이 사용된다
  - 문서화가 매우 잘 되어 있고, Visual Studio 등 개발 도구 지원이 훌륭하다
  - Windows와의 깊은 통합으로 최적화가 잘 되어 있다

- **단점**:
  - Windows와 Xbox에서만 사용 가능하다 (크로스 플랫폼 불가)
  - DirectX 12는 난이도가 높아 학습 곡선이 가파르다

### 2.3. Vulkan (Next-Gen & Cross-Platform)

- **특징**: OpenGL을 만든 크로노스 그룹에서 "OpenGL의 한계를 넘자"며 만든 차세대 API다.

- **장점**:
  - 크로스 플랫폼이면서도 DirectX 12급의 고성능을 제공한다
  - PC 게임을 만들어 안드로이드, 닌텐도 스위치 등으로 이식하기 용이하다
  - 멀티스레드를 완벽하게 지원하여 현대 멀티코어 CPU를 효율적으로 활용한다
  - GPU 제어를 세밀하게 할 수 있어 최적화 여지가 크다

- **단점**:
  - 진입 장벽이 매우 높다
  - 간단한 삼각형 하나를 그리는 데도 수백 줄의 초기화 코드가 필요하다
  - 메모리 관리, 동기화, 리소스 관리 등을 개발자가 직접 처리해야 한다
  - 실수하면 크래시나 예상치 못한 동작이 발생하기 쉽다

## 3. High-level vs Low-level API

API의 핵심적인 차이를 이해하는 또 다른 방법은 **추상화 수준**이다:

### High-level API (OpenGL, DirectX 11)

- 애플리케이션과 하드웨어 사이에 **두꺼운 드라이버 계층**이 존재한다
- 드라이버가 많은 것을 자동으로 처리해준다
- **장점**: 편하고 빠르게 개발 가능
- **단점**: 성능 오버헤드, 예측 불가능한 동작

### Low-level API (Vulkan, DirectX 12)

- 드라이버 계층이 **매우 얇다**
- 애플리케이션이 하드웨어(GPU)와 거의 직접 대화한다
- **장점**: 최대 성능, 완벽한 제어
- **단점**: 복잡하고 실수하기 쉬움

```text
High-level:  앱 → [두꺼운 드라이버] → GPU
Low-level:   앱 → [얇은 드라이버] → GPU
```

## 4. Vulkan, 왜 배워야 할까?

### 실제 사용 사례

Vulkan은 이미 많은 곳에서 사용되고 있다:

- **게임 엔진**:
  - Unreal Engine (옵션으로 지원)
  - Unity (옵션으로 지원)
  - id Tech 7 (DOOM Eternal)
  - Source 2 (Dota 2)

- **유명 게임**:
  - DOOM (2016), DOOM Eternal
  - Red Dead Redemption 2
  - Baldur's Gate 3
  - Valheim

- **기타**:
  - Android 게임 개발 (권장 API)
  - 에뮬레이터 (Dolphin, RPCS3)
  - 과학 시뮬레이션 및 연구

### 학습 가치

Vulkan을 배우면:

1. **GPU 작동 원리**를 깊이 이해하게 된다
2. **현대 게임 엔진**의 렌더링 파이프라인을 이해할 수 있다
3. **성능 최적화** 능력이 크게 향상된다
4. **크로스 플랫폼** 개발 역량을 얻는다

### 주의사항

- **그래픽 프로그래밍 초보자**라면 OpenGL을 먼저 배우는 것을 강력히 권장한다
- Vulkan은 **이미 그래픽 기초**가 있는 사람을 위한 API다
- 간단한 프로젝트라면 OpenGL이나 게임 엔진 사용이 더 효율적이다
